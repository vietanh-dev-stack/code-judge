Debug & xác định lỗi theo container

Stack production: docker-compose.production.yml. Mục tiêu: biết container nào gây lỗi và lệnh nào để kiểm tra.

Bắt đầu debug (3 bước)

  1. SSH vào VPS, vào thư mục repo: cd /root/code-judge
  2. Chạy lệnh docker compose ... ps (xem prefix) — tìm container Exit, unhealthy, hoặc không có trong danh sách
  3. Đối chiếu triệu chứng với bảng triệu chứng hoặc mở mục container tương ứng
Container migrate là one-shot: trạng thái Exited (0) là bình thường sau khi thành công. Exited (1) = lỗi — thường chặn core-api khởi động.

Thứ tự phụ thuộc (container nào chặn container nào)

app-db (healthy) ──┬──► migrate (phải completed) ──► core-api (healthy) app-redis (healthy) ─┘ │ minio (started) ────────────────────────────────────┘ ├──► web (healthy) ──► nginx └──► worker (cần judge0-server started) judge0-db + judge0-redis (healthy) ──► judge0-server + judge0-worker
Nếu core-api không lên → kiểm tra migrate trước, rồi app-db / app-redis.
Nếu nginx không lên → thường do web hoặc core-api chưa healthy, hoặc nginx thiếu cert SSL.
Nếu submit ALGO không chấm → worker + judge0-server, không phải nginx.

Prefix — chạy trước mọi lệnh trên VPS

Không dùng biến $COMPOSE rồi gõ $COMPOSE ps — trên một số shell biến trống sẽ gọi nhầm lệnh ps hệ thống và báo lỗi unsupported option. Luôn gõ đầy đủ lệnh docker compose -f ... như các khối bên dưới. 
cd /root/code-judge

Đổi /root/code-judge thành /home/ubuntu/code-judge nếu bạn deploy với user ubuntu.

Lưu ý env: docker compose restart không đọc lại .env.production mới. Sau khi sửa env dùng up -d --force-recreate <service>.

Lệnh chung (chạy trên VPS)

docker compose ps

Khi nào: Bước đầu — xem container nào running / healthy / exited.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps

docker compose ps -a

Khi nào: Cần thấy cả container đã exit (đặc biệt migrate exit 1).

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps -a

logs <service> --tail N

Khi nào: Đã biết service nghi ngờ — xem lỗi gần nhất.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production logs migrate --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs core-api --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs nginx --tail 50
docker compose -f docker-compose.production.yml --env-file .env.production logs worker --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-worker --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-server --tail 80

docker logs cj-prod-<tên>

Khi nào: Tương đương compose logs khi bạn nhớ tên container (bảng trên).

docker logs cj-prod-migrate 2>&1 | tail -50

docker inspect — health

Khi nào: Container running nhưng unhealthy — xem healthcheck fail vì sao.

docker inspect cj-prod-core-api --format '{{.State.Health.Status}}'
docker inspect cj-prod-core-api --format '{{json .State.Health}}' | head -c 2000

curl từ trong VPS (bỏ qua nginx / qua nginx)

Khi nào: Phân tách lỗi nginx vs app — gọi trực tiếp service nội bộ hoặc qua proxy.

# API qua nginx (localhost)
curl -s http://127.0.0.1:8080/tags | head
curl -sI https://api.code-judge.io.vn/tags | head -5

# Web qua nginx
curl -sI http://127.0.0.1/ | head -5
curl -sI https://code-judge.io.vn/ | head -5

# MinIO (host port)
curl -sI http://127.0.0.1:9000/minio/health/live | head -5

exec vào container (tùy chọn)

Khi nào: Kiểm tra sâu — psql, redis-cli, cấu hình nginx trong container.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production exec app-db psql -U codejudge -d codejudge -c 'SELECT 1;'
docker compose -f docker-compose.production.yml --env-file .env.production exec app-redis redis-cli ping
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -t

Kiểm tra env trong container (sau khi đổi .env)

Khi nào: Login/cookie/mail/MinIO vẫn sai sau khi sửa .env.production — xác nhận container đã nhận biến mới (cần force-recreate, không chỉ restart).

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production config | grep -E 'COOKIE_DOMAIN|MINIO_PUBLIC|MAIL_ACCOUNT'
docker compose -f docker-compose.production.yml --env-file .env.production exec core-api printenv COOKIE_DOMAIN MINIO_PUBLIC_BASE_URL MAIL_ACCOUNT

Debug từng container

migrate — cj-prod-migrate

Vai trò: Chạy prisma migrate deploy (+ seed nếu không RUN_SEED=0). One-shot; exit 1 chặn core-api.

Dấu hiệu lỗi ở đây: Compose báo migrate failed; P1000, P3015 trong log.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production logs migrate --tail 120
docker compose -f docker-compose.production.yml --env-file .env.production run --rm migrate

Không phải lỗi migrate nếu: Trang web load được và API /tags 200 — migrate có thể đã chạy xong trước đó (container exited 0).

app-db — cj-prod-app-db

Vai trò: Postgres cho Prisma / API / worker.

Dấu hiệu: migrate P1000; core-api crash kết nối DB; unhealthy nếu pg không ready.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps app-db
docker compose -f docker-compose.production.yml --env-file .env.production logs app-db --tail 40
docker compose -f docker-compose.production.yml --env-file .env.production exec -T app-db psql -U codejudge -d codejudge -c 'SELECT 1;'

core-api — cj-prod-core-api

Vai trò: REST API, auth, Socket.io, presigned MinIO URL.

Dấu hiệu: 502 trên api.*; CORS; /auth/me 401 sau login; cookie không có Domain=.your-domain; API JSON lỗi; healthcheck fail (/tags).

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps core-api
docker compose -f docker-compose.production.yml --env-file .env.production logs core-api --tail 80
docker inspect cj-prod-core-api --format '{{.State.Health.Status}}'
docker compose -f docker-compose.production.yml --env-file .env.production exec core-api printenv COOKIE_DOMAIN FRONTEND_URL MINIO_PUBLIC_BASE_URL

Env quan trọng: COOKIE_DOMAIN=.code-judge.io.vn (dấu chấm đầu = cookie dùng chung web + api subdomain); FRONTEND_URL gồm https://domain và https://www.domain nếu dùng www; MINIO_PUBLIC_BASE_URL=https://code-judge.io.vn (không :9000).

Sau khi sửa env:

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate core-api

Xóa cookie cũ trên trình duyệt (hoặc tab ẩn danh), đăng nhập lại. DevTools → Application → Cookies: phải thấy Domain=.code-judge.io.vn.

Phụ thuộc: migrate success, app-db, app-redis healthy, minio started.

web — cj-prod-web

Vai trò: Next.js; URL API nhúng lúc build (NEXT_PUBLIC_CORE_URL).

Dấu hiệu: Trang trắng / 502 qua nginx; Mixed Content gọi :8080; UI cũ sau đổi env API.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps web
docker compose -f docker-compose.production.yml --env-file .env.production logs web --tail 50

Phụ thuộc: core-api healthy. Sửa URL API → rebuild web, không chỉ restart.

nginx — cj-prod-nginx

Vai trò: Cổng 80/443 (web), 8080/443 api.*, SSL, redirect HTTP→HTTPS.

Dấu hiệu: Không vào được domain; thiếu :443 trong docker ps; SSL certificate error; 502 (upstream web/core-api down).

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps nginx
docker ps --filter name=cj-prod-nginx
docker compose -f docker-compose.production.yml --env-file .env.production logs nginx --tail 50
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -t
ls -la /etc/letsencrypt/live/code-judge.io.vn/

MinIO qua HTTPS: nginx proxy /codejudge/ → minio:9000 (xem deploy/nginx/conf.d/code-judge.conf). Sau sửa conf:

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate nginx
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -s reload

Phụ thuộc: web + core-api healthy. Cert thiếu → container có thể không start listen 443.

worker — cj-prod-worker

Vai trò: Queue chấm bài, gọi Judge0 / Lambda.

Dấu hiệu: Submit ALGO pending mãi; web/API OK nhưng không có kết quả chấm.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps worker
docker compose -f docker-compose.production.yml --env-file .env.production logs worker --tail 100

Phụ thuộc: core-api healthy, judge0-server started, app-db, app-redis, minio.

judge0-* — cj-prod-judge0-server / worker / db / redis

Vai trò: Chấm code ALGO. Production mặc định JUDGE0_USE_CGROUP=false + mount scripts/isolate_stub.sh (Ubuntu 24.04 cgroup v2 thường không chạy isolate thật).

Dấu hiệu: Internal Error, stdout null; Cannot write /sys/fs/cgroup/memory/box-*; sudo: Permission denied; No such file - isolate.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps judge0-server judge0-worker judge0-db judge0-redis
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-server --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-worker --tail 80
curl -s http://127.0.0.1:2358/languages | head

Sau sync từ Windows (CRLF / quyền thực thi stub):

cd /root/code-judge
sed -i 's/\r$//' scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x scripts/isolate_stub.sh scripts/sudo_stub.sh
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate judge0-server judge0-worker

Test stub trong container worker:

docker compose -f docker-compose.production.yml --env-file .env.production exec judge0-worker /usr/local/bin/isolate --init
docker compose -f docker-compose.production.yml --env-file .env.production exec judge0-worker ls -la /usr/local/bin/isolate /usr/bin/isolate

Env: JUDGE0_SECRET_KEY_BASE, JUDGE0_DB_PASSWORD — không ảnh hưởng migrate app-db.

Quy trình khoanh vùng nhanh

  1. docker compose -f docker-compose.production.yml --env-file .env.production ps -a — ai exit / unhealthy?
  2. Nếu migrate exit 1 → logs migrate (dừng ở đây, sửa DB/migration trước)
  3. Nếu core-api unhealthy → logs core-api; kiểm migrate đã OK chưa
  4. Nếu web unhealthy → logs web; core-api phải healthy trước
  5. Nếu nginx không có 443 → cert + logs nginx
  6. Login/dashboard → printenv COOKIE_DOMAIN trong core-api; cookie trình duyệt
  7. Upload/ảnh Mixed Content → MINIO_PUBLIC_BASE_URL + force-recreate core-api
  8. Nếu ps đều OK nhưng user lỗi → curl + DevTools → bảng trình duyệt
  9. Chỉ lỗi chấm bài → chmod stub scripts → logs worker + logs judge0-worker
