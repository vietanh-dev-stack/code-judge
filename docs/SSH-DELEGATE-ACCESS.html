Thêm quyền quản lý VPS — SSH key của người khác

Hướng dẫn cấp cho monitor / người vận hành quyền SSH vào VPS Code Judge (sync code, xem log Docker, cập nhật deploy) mà không chia sẻ private key của admin. Bổ sung cho mục SSH lần đầu trong DEPLOY-VPS.html.

Tổng quan

Sau khi VPS đã deploy (thường admin dùng root và thư mục /root/code-judge), bạn có thể thêm một hoặc nhiều người cùng quản lý bằng cách đưa public key của họ vào ~/.ssh/authorized_keys trên VPS.

Không bao giờ gửi file private (id_ed25519, không có đuôi .pub) qua chat, email hay Git. Mỗi người tự tạo cặp key trên máy mình.

Luồng tóm tắt

Monitor (Windows) Admin (đã có SSH) VPS | | | | ssh-keygen (tạo cặp key) | | | gửi file *.pub ------------------->| | | | ssh vào VPS | | | append vào authorized_keys | | | | ssh VPS_IP (dùng key riêng) <------|--------------------------| | sync / docker logs / deploy | |

Bước 1 — Monitor tạo SSH key (Windows)

Windows Người được cấp quyền chạy trên máy của họ:

ssh-keygen -t ed25519 -C "ten-monitor@cong-ty.example"

Nhấn Enter để dùng đường dẫn mặc định C:\Users\<TEN>\.ssh\id_ed25519. Có thể đặt passphrase (khuyến nghị).

Xem public key (một dòng bắt đầu bằng ssh-ed25519):

Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub
Nếu monitor đã có key khác, có thể tạo thêm với tên file riêng: ssh-keygen -t ed25519 -f $env:USERPROFILE\.ssh\id_ed25519_codejudge -C "monitor-codejudge"

Bước 2 — Gửi public key cho admin

Monitor gửi cho admin chỉ nội dung file *.pub (một dòng), qua kênh tin cậy (chat nội bộ, ticket, v.v.).

Ví dụ dòng public key (minh họa — không dùng key thật từ doc):

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx ten-monitor@example.com
Admin không cần và không được yêu cầu monitor gửi file id_ed25519 (private) hoặc passphrase.

Bước 3 — Admin thêm key lên VPS

VPS Admin SSH vào VPS bằng key hiện có, rồi chọn một trong hai cách dưới đây.

3.1. Cùng user deploy (khuyến nghị: root)

Dùng khi stack nằm tại /root/code-judge và monitor cần cùng quyền vận hành.

Cách A — Từ Windows của admin (một lệnh)

Khi nào: Admin có file monitor.pub trên máy Windows (monitor đã gửi nội dung, admin lưu thành file).

Windows

# Thay VPS_IP; file .pub do monitor cung cấp
type C:\path\to\monitor.pub | ssh root@VPS_IP "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

Cách B — Trực tiếp trên VPS

Khi nào: Admin đang SSH sẵn trên VPS.

VPS

mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys
# Dán thêm MỘT DÒNG public key của monitor ở cuối file (mỗi key một dòng)
chmod 600 ~/.ssh/authorized_keys

Kiểm tra số dòng key (mỗi dòng = một key được phép):

wc -l ~/.ssh/authorized_keys
grep -c '^ssh-' ~/.ssh/authorized_keys

3.2. User Linux riêng (tùy chọn)

Dùng khi muốn tách tài khoản, repo ở /home/ubuntu/code-judge:

# Trên VPS — admin chạy với sudo/root
adduser monitorops
usermod -aG docker monitorops
mkdir -p /home/monitorops/.ssh
chmod 700 /home/monitorops/.ssh
nano /home/monitorops/.ssh/authorized_keys
# Dán public key monitor
chmod 600 /home/monitorops/.ssh/authorized_keys
chown -R monitorops:monitorops /home/monitorops/.ssh

Monitor kết nối: ssh monitorops@VPS_IP. Cần đồng bộ code vào /home/monitorops/code-judge hoặc cấp quyền đọc thư mục deploy hiện có.

Bước 4 — Monitor kiểm tra kết nối

Windows Trên máy monitor (sau khi admin báo đã thêm key):

ssh root@VPS_IP
# hoặc: ssh ubuntu@VPS_IP

Nếu dùng key không mặc định:

ssh -i $env:USERPROFILE\.ssh\id_ed25519_codejudge root@VPS_IP

Trên VPS, xác nhận vào đúng thư mục project:

cd /root/code-judge
ls docker-compose.production.yml .env.production
docker compose -f docker-compose.production.yml --env-file .env.production ps
Thấy danh sách container Up → SSH và quyền Docker ổn. Chi tiết deploy: UPDATE-DEPLOY.html.

Việc monitor thường làm sau khi có quyền

  • SSH xem log: docker compose ... logs core-api --tail 100
  • Cập nhật code từ Windows (mục dưới)
  • Đọc DEBUG-CONTAINERS.html khi service lỗi

sync-to-vps.ps1

Windows Monitor chạy trên máy có clone repo code-judge:

cd C:\path\to\code-judge
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\sync-to-vps.ps1 `
  -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

OpenSSH trên Windows tự dùng %USERPROFILE%\.ssh\id_ed25519 (hoặc key trong config).

~/.ssh/config (tuỳ chọn, Windows)

File C:\Users\<TEN>\.ssh\config cho monitor:

Host codejudge-vps
    HostName VPS_IP
    User root
    Port 22
    IdentityFile ~/.ssh/id_ed25519
ssh codejudge-vps
scp .env.production codejudge-vps:/root/code-judge/.env.production
File .env.production chứa secret — chỉ copy qua SSH/SFTP tin cậy; trên VPS giữ chmod 600 .env.production.

Thu hồi quyền (gỡ key)

VPS Admin SSH vào VPS, mở ~/.ssh/authorized_keys, xóa đúng một dòng public key của monitor, lưu file.

nano ~/.ssh/authorized_keys
# xóa dòng key của monitor, lưu
chmod 600 ~/.ssh/authorized_keys

Monitor sẽ không SSH được nữa (trừ khi còn key/password khác). Không cần reboot VPS.

Nếu dùng user riêng: deluser --remove-home monitorops (cẩn thận — xóa home và dữ liệu local của user đó).

Bảo mật

  • Mỗi người một cặp key; rotate khi máy monitor bị mất hoặc nhân viên rời team (gỡ dòng key cũ, tạo key mới).
  • Giữ port 22 (hoặc port SSH tùy chỉnh) trong firewall; cân nhắc giới hạn IP nguồn trên panel VPS nếu team cố định IP văn phòng.
  • Không thêm key vào VPS qua kênh công khai (issue GitHub công khai, pastebin).
  • Quyền root = toàn quyền host + đọc .env.production; chỉ cấp cho người tin cậy.
  • Sau khi thêm user vào nhóm docker: monitor phải đăng xuất SSH và vào lại thì docker không cần sudo.
