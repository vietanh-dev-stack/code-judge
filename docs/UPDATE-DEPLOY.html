Cập nhật deploy — từng lệnh chi tiết

Mỗi lệnh kèm khi nào dùng, chạy trên Windows hay VPS. Stack: docker-compose.production.yml + .env.production.

Tổng quan & quyết định nhanh

Bạn vừa thay đổi gì?Làm gì (tóm tắt)
Chỉ .env.production (không đổi NEXT_PUBLIC_*) scp hoặc nano trên VPS → up -d --force-recreate service liên quan (không chỉ restart)
Judge0 không chấm / Internal Error Sync code → sed + chmod +x scripts/*.sh → force-recreate judge0-*
NEXT_PUBLIC_CORE_URL (URL API cho trình duyệt) Sửa env → build lại web → up -d web
Code TypeScript/React/API (không đụng prisma/migrations) Windows: sync-to-vps.ps1 → VPS: production-up.sh
Thêm/sửa thư mục trong prisma/migrations Sync code → kiểm tra migration.sql → build migrate → run --rm migrate → production-up.sh
Quan trọng: sync-to-vps.ps1 không đưa .env.production lên VPS. Env phải sửa trên VPS hoặc copy bằng scp riêng.

Chuẩn bị đường dẫn (VPS)

Thay /root/code-judge và IP nếu bạn dùng user ubuntu hoặc thư mục khác.

VPS cd /root/code-judge

Khi nào: Trước mọi lệnh docker compose trên VPS — phải đứng ở thư mục có file docker-compose.production.yml.

cd /root/code-judge
Lệnh trong doc này đều gõ đầy đủ — không dùng $COMPOSE. Nếu biến COMPOSE chưa được export, gõ $COMPOSE ps có thể gọi nhầm lệnh ps của hệ thống.

Mẫu lệnh (mọi khối bên dưới đều bắt đầu bằng cd + docker compose -f docker-compose.production.yml --env-file .env.production):

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production ps

Lệnh trên Windows (PowerShell)

Windows cd vào thư mục repo

Khi nào: Trước khi chạy sync-to-vps.ps1 hoặc scp env — phải ở gốc monorepo (cùng cấp docker-compose.production.yml).

cd C:\Users\ADMIN\Documents\GitHub\code-judge

Windows sync-to-vps.ps1 (PowerShell)

Khi nào: Bạn đổi mã nguồn trên máy (API, web, worker, nginx, prisma, docker-compose, scripts stub, …). Không đưa .env.production — dùng scp riêng.

cd C:\Users\ADMIN\Documents\GitHub\code-judge
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\sync-to-vps.ps1 `
  -VpsHost 129.212.237.24 `
  -VpsUser root `
  -RemoteDir /root/code-judge

Mặc định script dùng ubuntu — VPS chỉ có root thì bắt buộc -VpsUser root -RemoteDir /root/code-judge. Chờ Sync done. Sau sync: sed + chmod trên VPS.

Windows scp .env.production

Khi nào: Bạn sửa .env.production trên máy Windows và muốn ghi đè file env trên VPS (không cần sync cả repo).

scp .env.production root@129.212.237.24:/root/code-judge/.env.production

Sau scp, trên VPS chạy lệnh áp dụng env theo mục env (restart / rebuild service).

Sửa .env.production trên VPS

VPS nano .env.production

Khi nào: Sửa secret/URL trực tiếp trên server (không copy từ Windows).

cd /root/code-judge
nano .env.production

VPS chmod 600 .env.production

Khi nào: Sau khi tạo hoặc copy env — chỉ user owner đọc được (bảo mật).

chmod 600 .env.production

Lệnh theo loại env đổi

Đổi NEXT_PUBLIC_CORE_URL (URL API cho trình duyệt)

Ví dụ: từ http://...:8080 sang https://api.code-judge.io.vn — tránh mixed content trên HTTPS.

VPS build web --no-cache

Khi nào: Sau khi sửa NEXT_PUBLIC_CORE_URL trong .env.production. Biến này được nhúng lúc build image Next.js — restart thường không đủ.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production build web --no-cache

VPS up -d web nginx

Khi nào: Ngay sau build web — chạy container web mới và nginx (proxy).

docker compose -f docker-compose.production.yml --env-file .env.production up -d web nginx

VPS ./deploy/production-up.sh (thay cho build web riêng)

Khi nào: Không chắc service nào cần rebuild — script build tất cả image và up -d --build (an toàn, lâu hơn).

cd /root/code-judge
./deploy/production-up.sh

Đổi FRONTEND_URL, COOKIE_DOMAIN, JWT_*, MAIL_*, GOOGLE_*, MINIO_PUBLIC_BASE_URL

Các biến này được core-api đọc lúc khởi tạo container. Sau khi sửa .env.production dùng --force-recreate — restart thường không nạp env mới.

Ví dụ HTTPS subdomain (thay domain của bạn):

FRONTEND_URL=https://code-judge.io.vn,https://www.code-judge.io.vn
COOKIE_DOMAIN=.code-judge.io.vn
MINIO_PUBLIC_BASE_URL=https://code-judge.io.vn
MAIL_ACCOUNT=your@gmail.com
MAIL_PASSWORD=xxxx-xxxx-xxxx-xxxx

MAIL_PASSWORD = Gmail App Password (16 ký tự), không phải mật khẩu đăng nhập Gmail thường.

Windows scp .env.production lên VPS

Khi nào: Sửa env trên máy dev, cần ghi đè file trên VPS.

cd C:\Users\ADMIN\Documents\GitHub\code-judge
scp .env.production root@129.212.237.24:/root/code-judge/.env.production

VPS force-recreate core-api

Khi nào: Sau khi sửa/scp env (CORS, cookie, mail, JWT, MinIO public URL).

cd /root/code-judge
chmod 600 .env.production
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate core-api
docker compose -f docker-compose.production.yml --env-file .env.production exec core-api printenv COOKIE_DOMAIN MINIO_PUBLIC_BASE_URL MAIL_ACCOUNT

VPS Kiểm tra compose đã map env

Khi nào: printenv trong container trống nhưng file .env đã có giá trị — xác nhận bước recreate đã chạy.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production config | grep -E 'COOKIE_DOMAIN|MINIO_PUBLIC|MAIL_'

VPS Sau đổi MINIO_PUBLIC_BASE_URL — reload nginx

Khi nào: Upload/ảnh Mixed Content — cần proxy /codejudge/ trong nginx (đã có trong repo) và URL public HTTPS.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate nginx
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -t
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -s reload

Đổi POSTGRES_PASSWORD / POSTGRES_USER

VPS Kiểm tra psql vào DB

Khi nào: Migrate báo P1000 hoặc sau khi đổi mật khẩu trong env — kiểm tra user có vào được DB không.

docker compose -f docker-compose.production.yml --env-file .env.production exec -T app-db \
  psql -U codejudge -d codejudge -c 'SELECT 1;'

Nếu lỗi authentication → mật khẩu trong .env không khớp volume Postgres cũ.

VPS ALTER USER (đồng bộ mật khẩu DB với .env)

Khi nào: Bạn đổi POSTGRES_PASSWORD trong env nhưng giữ dữ liệu volume — phải đặt lại mật khẩu user trong Postgres cho khớp env (thay MAT_KHAU_TRONG_ENV bằng giá trị thật trong file).

docker compose -f docker-compose.production.yml --env-file .env.production exec -T app-db \
  psql -U codejudge -d codejudge \
  -c "ALTER USER codejudge WITH PASSWORD 'MAT_KHAU_TRONG_ENV';"

VPS down -v (xóa DB — chỉ khi chấp nhận mất data)

Khi nào: Môi trường test, không cần giữ DB; muốn Postgres init lại với POSTGRES_PASSWORD mới trong env. Cảnh báo: xóa volume → mất toàn bộ dữ liệu app DB.

docker compose -f docker-compose.production.yml --env-file .env.production down -v
./deploy/production-up.sh

Đổi MINIO_ROOT_* / MINIO_PUBLIC_BASE_URL

Production HTTPS: MINIO_PUBLIC_BASE_URL=https://code-judge.io.vn (cùng domain web, path /codejudge/ qua nginx). Không dùng http://IP:9000 trên trang HTTPS.

VPS force-recreate minio core-api worker

Khi nào: Đổi credential MinIO hoặc URL public presigned.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate minio core-api worker

Judge0 — stub isolate sau sync code

Ubuntu 24.04 production dùng JUDGE0_USE_CGROUP=false và mount scripts/isolate_stub.sh. Sau sync từ Windows, file .sh có thể mất quyền thực thi hoặc CRLF.

VPS sed + chmod stub scripts

Khi nào: Log judge0: sudo: Permission denied, No such file - isolate, hoặc submit Internal Error.

cd /root/code-judge
sed -i 's/\r$//' scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x scripts/isolate_stub.sh scripts/sudo_stub.sh
ls -la scripts/isolate_stub.sh scripts/sudo_stub.sh

VPS force-recreate judge0-server judge0-worker

Khi nào: Sau chmod/sed hoặc sau khi sync docker-compose.production.yml mới (mount /usr/bin/isolate).

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate judge0-server judge0-worker
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-worker --tail 40
docker compose -f docker-compose.production.yml --env-file .env.production exec judge0-worker /usr/local/bin/isolate --init

VPS Test API Judge0

Khi nào: Kiểm tra judge0-server đã listen (port nội bộ Docker, không publish ra host).

curl -s http://127.0.0.1:2358/languages | head

Nếu connection refused — container judge0-server chưa healthy; xem logs judge0-server.

Đổi HTTP_PORT / API_HTTP_PORT / HTTPS_PORT

VPS up -d nginx

Khi nào: Đổi map cổng host (80, 8080, 443) trong env. Nhớ mở cổng tương ứng trên UFW và DigitalOcean Cloud Firewall.

docker compose -f docker-compose.production.yml --env-file .env.production up -d nginx

Chỉ đổi env “nhẹ” (không đụng NEXT_PUBLIC_*, không đổi code)

VPS up -d (cả stack)

Khi nào: Đổi vài biến env, muốn Docker áp dụng lại cấu hình cho các service (không rebuild image). Nhanh nhưng vẫn có thể trigger migrate nếu compose phụ thuộc.

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d

Cập nhật code (không đổi prisma/migrations)

Luồng: Windows sync → VPS build & up.

Windows sync-to-vps.ps1

Khi nào: Sửa file trong repo (API, web, worker, Dockerfile, nginx conf, …) trên máy dev.

cd C:\Users\ADMIN\Documents\GitHub\code-judge
.\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

VPS chmod stub + ./deploy/production-up.sh

Khi nào: Sau sync — build lại image và khởi động stack. Nên chmod stub trước nếu có Judge0.

cd /root/code-judge
sed -i 's/\r$//' scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x deploy/production-up.sh deploy/production-down.sh
./deploy/production-up.sh

VPS build + up chỉ core-api

Khi nào: Chỉ sửa code backend, không đổi migration, không đổi NEXT_PUBLIC_* — tiết kiệm thời gian.

docker compose -f docker-compose.production.yml --env-file .env.production build core-api
docker compose -f docker-compose.production.yml --env-file .env.production up -d core-api

VPS build + up chỉ web

Khi nào: Chỉ sửa frontend Next.js (và/hoặc đã đổi NEXT_PUBLIC_CORE_URL trong env).

docker compose -f docker-compose.production.yml --env-file .env.production build web
docker compose -f docker-compose.production.yml --env-file .env.production up -d web

VPS build + up chỉ worker

Khi nào: Chỉ sửa logic chấm bài / queue worker.

docker compose -f docker-compose.production.yml --env-file .env.production build worker
docker compose -f docker-compose.production.yml --env-file .env.production up -d worker

VPS up -d nginx (sau khi đổi deploy/nginx/conf.d)

Khi nào: Chỉ sửa cấu hình nginx (SSL, redirect, proxy) — không cần rebuild app.

docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate nginx

Cập nhật thư mục prisma/migrations

Dùng khi thêm migration mới, sửa migration.sql, hoặc sync migrations từ máy dev.

Windows sync-to-vps.ps1

Khi nào: Thư mục apps/core-api/prisma/migrations đổi trên máy dev — phải đẩy lên VPS trước khi chạy migrate.

.\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

VPS find migration.sql

Khi nào: Trước migrate — đảm bảo mỗi thư mục 20xxxxxxxx_* có file migration.sql. Tránh lỗi P3015.

cd /root/code-judge
find apps/core-api/prisma/migrations -name migration.sql -print

VPS ls từng thư mục migration

Khi nào: Nghi ngờ có thư mục migration rỗng / thiếu file (hay gặp sau sync lỗi).

for d in apps/core-api/prisma/migrations/20*; do echo "==== $d"; ls -la "$d"; done

VPS rm -rf thư mục migration không có .sql

Khi nào: Thư mục con không có migration.sql và là thư mục tạo nhầm — không xóa nếu DB đã ghi migration đó trong _prisma_migrations.

rm -rf apps/core-api/prisma/migrations/20260519073615_db_v1
rm -rf apps/core-api/prisma/migrations/20260520120000_user_profile_fields

Ví dụ tên thư mục — thay bằng tên thật trên VPS của bạn.

VPS RUN_SEED=0 trong .env

Khi nào: DB production đã có dữ liệu thật — tránh prisma db seed ghi đè sau migrate.

nano .env.production
# thêm hoặc sửa dòng:
RUN_SEED=0

VPS build migrate

Khi nào: Sau sync migrations mới — build image one-shot chứa thư mục prisma/migrations mới nhất.

docker compose -f docker-compose.production.yml --env-file .env.production build migrate

VPS run --rm migrate

Khi nào: Chạy prisma migrate deploy (+ seed nếu không có RUN_SEED=0) một lần; in log ra terminal để debug.

docker compose -f docker-compose.production.yml --env-file .env.production run --rm migrate

VPS logs migrate

Khi nào: Job migrate fail trong up -d — xem lỗi P1000, P3015, seed, …

docker compose -f docker-compose.production.yml --env-file .env.production logs migrate --tail 120

VPS production-up.sh (sau migrate OK)

Khi nào: Migrate thành công — khởi động lại core-api, worker và các service phụ thuộc.

./deploy/production-up.sh

Redeploy sau conflict migration (khuyến nghị)

Triệu chứng: P3018 / type "Role" already exists — DB đã apply migration cũ (migrate_v11) nhưng repo chỉ còn db_v1; hoặc VPS còn thư mục migration cũ sau sync.

Windows sync + scp env

Khi nào: Đẩy code mới và cập nhật .env.production lên VPS.

cd C:\Users\ADMIN\Documents\GitHub\code-judge
.\deploy\sync-to-vps.ps1 -VpsHost 129.212.237.24 -VpsUser root -RemoteDir /root/code-judge
scp .env.production root@129.212.237.24:/root/code-judge/.env.production

VPS fix-migration-rename-vps.sh (một lần khi conflict)

Khi nào: Migrate fail vì schema đã có từ migration cũ — sửa _prisma_migrations, rebuild image migrate.

cd /root/code-judge
chmod +x deploy/fix-migration-rename-vps.sh
./deploy/fix-migration-rename-vps.sh

VPS redeploy-vps.sh

Khi nào: Sau fix migration (hoặc migrate bình thường) — build web không cache nếu đổi NEXT_PUBLIC_*.

RUN_SEED=0 BUILD_WEB_NOCACHE=1 ./deploy/redeploy-vps.sh
# Nếu vừa chạy fix-migration-rename-vps.sh:
RUN_SEED=0 BUILD_WEB_NOCACHE=1 SKIP_MIGRATE=1 ./deploy/redeploy-vps.sh

Kiểm tra sau cập nhật

VPS docker compose ps

Khi nào: Sau mọi lần up — xem service nào running/healthy, migrate đã completed chưa.

docker compose -f docker-compose.production.yml --env-file .env.production ps

VPS curl API tags

Khi nào: Kiểm tra API sống (HTTP qua localhost hoặc HTTPS qua domain).

curl -s http://127.0.0.1:8080/tags | head
curl -sI https://api.code-judge.io.vn/tags | head -5

VPS curl HTTPS web

Khi nào: Sau đổi nginx SSL hoặc deploy web.

curl -sI https://code-judge.io.vn | head -5

VPS docker logs (khi lỗi)

Khi nào: Service unhealthy hoặc 502/401 — xem log từng service.

docker compose -f docker-compose.production.yml --env-file .env.production logs core-api --tail 50
docker compose -f docker-compose.production.yml --env-file .env.production logs nginx --tail 30
docker compose -f docker-compose.production.yml --env-file .env.production logs migrate --tail 50

Lỗi thường gặp & lệnh sửa

LỗiNguyên nhânLệnh / hành động
P1000 POSTGRES_PASSWORD env ≠ mật khẩu trong volume ALTER USER trong app-db hoặc khớp lại env; xem mục Postgres
P3015 Thư mục migration thiếu migration.sql find + sync lại + rm -rf thư mục rỗng + build migrate
Mixed Content Web HTTPS gọi API HTTP (:8080) Sửa NEXT_PUBLIC_CORE_URL=https://api... → build web --no-cache → up -d web
CORS FRONTEND_URL ≠ origin trình duyệt Sửa env → up -d core-api
401 /auth/me sau login Thiếu COOKIE_DOMAIN trong container; cookie cũ; web gọi sai API COOKIE_DOMAIN=.domain → force-recreate core-api; xóa cookie; kiểm NEXT_PUBLIC_CORE_URL
Mixed Content :9000 MINIO_PUBLIC_BASE_URL trỏ HTTP :9000 Đặt https://domain + nginx /codejudge/ → recreate core-api + reload nginx
Judge0 Internal Error Stub không executable / CRLF sed + chmod +x scripts/*.sh → recreate judge0-*
Mail không gửi MAIL_* chưa vào container scp env → force-recreate core-api; dùng Gmail App Password
Connection refused :443 Nginx chưa map 443 hoặc chưa cert Sync compose mới → certbot → up -d --force-recreate nginx
