Triển khai Code Judge lên VPS

Ubuntu 24.04 LTS x64 Docker Compose production Cập nhật: 2026-05-18

Hướng dẫn đầy đủ: kết nối SSH từ Windows → tạo .env.production từng biến → build và chạy toàn bộ stack trên VPS bằng docker-compose.production.yml. Không cần cài Node/PM2 trên host (container tự build).

2. Yêu cầu

VPS

  • OS: Ubuntu 24.04 LTS, kiến trúc x86_64 (amd64)
  • RAM: tối thiểu 4 GB (khuyến nghị 8 GB — Judge0 + Postgres)
  • Disk: ≥ 20 GB
  • Port mở: 22 (SSH), 80 (web), 8080 (API khi chỉ dùng IP), sau này 443 (HTTPS)

Máy Windows (máy bạn)

  • OpenSSH (ssh, scp) hoặc PuTTY + WinSCP
  • Biết VPS_IP, VPS_USER (thường ubuntu trên Ubuntu cloud)

3. Kết nối SSH từ Windows

Mọi lệnh trong mục 4–8 chạy trên VPS sau khi SSH thành công, trừ khi ghi rõ “trên Windows”.

3.1. Tạo SSH key (khuyến nghị)

Trên Windows — PowerShell:

ssh-keygen -t ed25519 -C "your-email@example.com"

Key lưu tại C:\Users\<TEN_BAN>\.ssh\id_ed25519 (private) và id_ed25519.pub (public).

Copy public key lên VPS (lần đầu có thể dùng password):

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh ubuntu@VPS_IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys"

3.2. Kết nối

ssh ubuntu@VPS_IP

Port SSH khác 22:

ssh -p 2222 ubuntu@VPS_IP

3.3. Cấu hình ~/.ssh/config (tuỳ chọn)

File C:\Users\<TEN_BAN>\.ssh\config:

Host codejudge-vps
    HostName VPS_IP
    User ubuntu
    Port 22
    IdentityFile ~/.ssh/id_ed25519
ssh codejudge-vps

3.4. PuTTY / WinSCP

  • PuTTY: Host VPS_IP, port 22, SSH — đăng nhập user ubuntu.
  • WinSCP: SFTP, cùng host/user — dùng kéo thả file .env.production lên VPS (mục 5).

4. Chuẩn bị VPS lần đầu (Ubuntu 24.04)

Trên VPS (đã SSH vào):

cd ~
git clone https://github.com/<ORG>/code-judge.git
cd code-judge
chmod +x deploy/ubuntu-24.04-setup.sh deploy/production-up.sh
./deploy/ubuntu-24.04-setup.sh

Script trên sẽ:

  • Cài Docker Engine + Compose plugin
  • Bật UFW: 22, 80, 8080
  • Tạo swap 2 GB nếu RAM < 6 GB
Sau khi cài Docker, đăng xuất SSH và vào lại để user được nhóm docker (không cần sudo mỗi lần docker compose).

Kiểm tra:

docker --version
docker compose version
lsb_release -a   # phải thấy Ubuntu 24.04

Không cần cài Node.js hay PM2 nếu deploy bằng Docker production.

6. File .env.production trên VPS

6.1. Tạo và chỉnh file trên VPS

Trên VPS, trong thư mục repo:

cd ~/code-judge
cp .env.production.example .env.production
nano .env.production

Hoặc chỉnh trên Windows rồi upload:

scp C:\path\to\.env.production ubuntu@VPS_IP:/home/ubuntu/code-judge/.env.production

Bảo vệ quyền file:

chmod 600 ~/code-judge/.env.production
Git không commit .env.production (đã có trong .gitignore). Mỗi lần đổi secret hoặc URL public, sửa file trên VPS rồi chạy lại ./deploy/production-up.sh. Đổi NEXT_PUBLIC_CORE_URL bắt buộc build lại image web (script up đã có --build).

6.2. Sinh secret ngẫu nhiên trên VPS

# JWT (32+ ký tự hex)
openssl rand -hex 32

# Judge0 Rails secret (64+ ký tự hex)
openssl rand -hex 64

# Mật khẩu Postgres / MinIO (ví dụ 24 ký tự base64)
openssl rand -base64 24

Dán giá trị vào đúng dòng trong .env.production.

6.3. Hai kịch bản URL

A — Có domain + DNS

Trỏ bản ghi A:

  • your-domain.com → VPS_IP
  • api.your-domain.com → VPS_IP
FRONTEND_URL=https://your-domain.com
NEXT_PUBLIC_CORE_URL=https://api.your-domain.com
GOOGLE_CALLBACK_URL=https://api.your-domain.com/auth/google/callback

User mở web: https://your-domain.com (sau khi gắn HTTPS — mục 10).

B — Chỉ có IP (test nhanh)

FRONTEND_URL=http://203.0.113.10
NEXT_PUBLIC_CORE_URL=http://203.0.113.10:8080
Dịch vụURL trình duyệt / client
Webhttp://VPS_IP/ (nginx port 80)
APIhttp://VPS_IP:8080/ (nginx port 8080)
Đừng dùng localhost trong FRONTEND_URL hay NEXT_PUBLIC_CORE_URL — user truy cập từ máy khác sẽ lỗi CORS / API.

8. Build và deploy trên VPS

Trên VPS — đảm bảo đã có .env.production hợp lệ:

cd ~/code-judge
./deploy/production-up.sh

Tương đương:

docker compose -f docker-compose.production.yml --env-file .env.production up -d --build

Thứ tự khởi động:

  1. app-db, app-redis, minio, Judge0 — healthy
  2. migrate — prisma migrate deploy (one-shot)
  3. core-api → worker → web → nginx

Theo dõi (Judge0 lần đầu 2–5 phút):

docker compose -f docker-compose.production.yml --env-file .env.production ps
docker compose -f docker-compose.production.yml --env-file .env.production logs -f judge0-server
docker compose -f docker-compose.production.yml --env-file .env.production logs -f core-api

8.1. Seed dữ liệu test (tuỳ chọn)

docker compose -f docker-compose.production.yml --env-file .env.production exec core-api \
  npx prisma db seed

Mật khẩu user seed: xem PRISMA-SEED.html. Không seed trên production thật nếu không cần.

8.2. Dừng / xoá stack

./deploy/production-down.sh
# Hoặc xoá cả volume (MẤT DỮ LIỆU):
# docker compose -f docker-compose.production.yml --env-file .env.production down -v

10. HTTPS (Let's Encrypt)

Nginx trong Docker đang listen 80/8080. Có hai hướng:

A — Certbot trên host + proxy host Nginx (phổ biến)

Cài Nginx/Certbot trên Ubuntu host, proxy tới 127.0.0.1:80 map từ container, hoặc publish web/api port khác. Sau khi có cert, cập nhật:

FRONTEND_URL=https://your-domain.com
NEXT_PUBLIC_CORE_URL=https://api.your-domain.com

Rồi ./deploy/production-up.sh để rebuild web.

B — Chỉ HTTP + IP (lab / nội bộ)

Dùng kịch bản mục 6.3.B — không cần certbot.

12. Xử lý lỗi thường gặp

Không SSH được

  • Kiểm tra security group / firewall nhà cung cấp (port 22).
  • Đúng user ubuntu và IP public.

permission denied khi chạy docker

sudo usermod -aG docker $USER
# logout SSH, login lại

Set POSTGRES_PASSWORD in .env.production

Thiếu biến bắt buộc — mở .env.production và set đủ các dòng có ?Set trong compose.

Web lỗi API / CORS

  • NEXT_PUBLIC_CORE_URL phải là URL API từ trình duyệt user (không phải http://core-api:3000).
  • FRONTEND_URL khớp URL web (VD http://IP không port nếu nginx :80).
  • Rebuild web sau khi sửa NEXT_PUBLIC_*.

Worker không chấm

docker compose -f docker-compose.production.yml --env-file .env.production logs worker --tail 80
docker compose -f docker-compose.production.yml --env-file .env.production logs judge0-server --tail 80

migrate failed

docker compose -f docker-compose.production.yml --env-file .env.production logs migrate

Thường do app-db chưa sẵn sàng hoặc POSTGRES_* sai.

13. Bảo mật

  • chmod 600 .env.production — chỉ owner đọc được.
  • Không mở port 5432, 6379, 9000, 2358 ra internet (compose production không publish các port này).
  • Đổi toàn bộ mật khẩu mặc định dev (codejudge, minioadmin).
  • Rotate JWT_SECRET nếu từng lộ.
  • Backup volume: app_postgres_data, minio_data.

Phụ lục: Deploy bằng PM2 (không Docker app)

Nếu chỉ chạy hạ tầng bằng Docker mà app chạy Node trực tiếp:

  1. docker compose up -d — file docker-compose.yml (dev infra)
  2. Tạo apps/core-api/.env, apps/worker/.env, apps/web/.env với localhost cho DB/Redis/MinIO/Judge0
  3. npm install → prisma migrate deploy → npm run build x3
  4. PM2 ecosystem.config.cjs (xem phiên bản doc cũ hoặc README)

Khuyến nghị VPS mới: dùng Docker production (mục 8) thay vì PM2.
