Triển khai Code Judge lên VPS

Ubuntu 24.04 LTS x64 Docker Compose production Cập nhật: 2026-05-20

Hướng dẫn đầy đủ: SSH từ Windows → đồng bộ mã nguồn lên VPS (không git clone GitHub) → .env.production → build stack docker-compose.production.yml trên VPS. Không cần Node/PM2 trên host (Docker build trong container).

Đánh giá DevOps (2026-05): Code Judge sẵn sàng deploy test trên VPS (Ubuntu 24.04 x64, RAM ≥ 4 GB). CI pass lint/test/build. Production compose gồm migrate, Judge0, nginx. Script: deploy/sync-to-vps.ps1, deploy/README.md.

1. Có thể deploy test trên VPS chưa?

Có. Repo đã có stack production hoàn chỉnh; build ứng dụng pass CI. Điều kiện:

Ubuntu 24.04 LTS x64Judge0 cần linux/amd64 + privileged
RAM ≥ 4 GB8 GB khuyến nghị; script tự tạo swap 2G nếu RAM < 6G

Tóm tắt nhanh: deploy/README.md trong repo.

4. Kết nối SSH từ Windows

Mọi lệnh trong mục 4–8 chạy trên VPS sau khi SSH thành công, trừ khi ghi rõ “trên Windows”.

3.1. Tạo SSH key (khuyến nghị)

Trên Windows — PowerShell:

ssh-keygen -t ed25519 -C "your-email@example.com"

Key lưu tại C:\Users\<TEN_BAN>\.ssh\id_ed25519 (private) và id_ed25519.pub (public).

Copy public key lên VPS (lần đầu có thể dùng password):

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh ubuntu@VPS_IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys"

3.2. Kết nối

ssh ubuntu@VPS_IP

Port SSH khác 22:

ssh -p 2222 ubuntu@VPS_IP

3.3. Cấu hình ~/.ssh/config (tuỳ chọn)

File C:\Users\<TEN_BAN>\.ssh\config:

Host codejudge-vps
    HostName VPS_IP
    User ubuntu
    Port 22
    IdentityFile ~/.ssh/id_ed25519
ssh codejudge-vps

3.4. PuTTY / WinSCP

  • PuTTY: Host VPS_IP, port 22, SSH — đăng nhập user ubuntu.
  • WinSCP: SFTP, cùng host/user — dùng kéo thả file .env.production lên VPS (mục 5).

5. Chuẩn bị VPS lần đầu (Ubuntu 24.04)

Lỗi cd ~/code-judge: No such file or directory? Thư mục chưa tồn tại — bạn chưa làm mục 6 (đồng bộ code từ Windows). Làm mục 6 trước, rồi quay lại mục 5.

Thứ tự bắt buộc:

  1. Mục 6 — trên Windows: sync-to-vps.ps1 (hoặc WinSCP) đưa code lên VPS
  2. Mục 5 — trên VPS: cd vào đúng thư mục → ubuntu-24.04-setup.sh

Đăng nhập bằng root (DigitalOcean / VPS của bạn)

Prompt root@ubuntu-... → thư mục repo là /root/code-judge, không phải /home/ubuntu/.

Trên Windows (thay VPS_IP bằng IP thật):

cd C:\Users\ADMIN\Documents\GitHub\code-judge
.\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

Trên VPS (sau khi sync xong):

cd /root/code-judge
ls deploy/ubuntu-24.04-setup.sh   # phải thấy file — nếu không, chưa sync
chmod +x deploy/ubuntu-24.04-setup.sh deploy/production-up.sh
./deploy/ubuntu-24.04-setup.sh

Nếu đăng nhập user ubuntu thì dùng cd ~/code-judge (= /home/ubuntu/code-judge) và sync với -VpsUser ubuntu.

Script trên sẽ:

  • Cài Docker Engine + Compose plugin
  • Bật UFW: 22, 80, 8080
  • Tạo swap 2 GB nếu RAM < 6 GB
Sau khi cài Docker, đăng xuất SSH và vào lại để user được nhóm docker (không cần sudo mỗi lần docker compose).

Kiểm tra:

docker --version
docker compose version
lsb_release -a   # phải thấy Ubuntu 24.04

Không cần cài Node.js hay PM2 nếu deploy bằng Docker production.

6. Đưa mã nguồn lên VPS (SSH — không GitHub)

Không dùng git clone / git pull từ GitHub. Chỉ đồng bộ từ máy dev (SCP, rsync, WinSCP, hoặc script).

6.1. Script PowerShell (khuyến nghị)

Trên Windows — mở PowerShell hoặc Windows Terminal (profile PowerShell), không dùng Command Prompt (cmd.exe): trong cmd, gõ .ps1 thường không chạy script và không có log.

Trong thư mục repo:

cd C:\Users\ADMIN\Documents\GitHub\code-judge
# User ubuntu (mặc định script):
.\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser ubuntu
# User root (DigitalOcean):
.\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

Nếu chỉ có cmd, gọi PowerShell rõ ràng:

cd C:\Users\ADMIN\Documents\GitHub\code-judge
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\sync-to-vps.ps1 -VpsHost VPS_IP -VpsUser root -RemoteDir /root/code-judge

Script đóng gói tar.gz (bỏ node_modules, .next, .env), upload qua scp, giải nén vào thư mục đích (mặc định /home/ubuntu/code-judge hoặc -RemoteDir bạn truyền).

6.2. WinSCP / SFTP

Kéo thả cả thư mục code-judge lên /home/ubuntu/code-judge. Bỏ qua: node_modules, apps/web/.next, file .env local.

6.3. rsync (WSL hoặc Git Bash)

cd /mnt/c/Users/ADMIN/Documents/GitHub/code-judge
rsync -avz --delete --exclude-from=deploy/rsync-exclude.txt \
  ./ ubuntu@VPS_IP:/home/ubuntu/code-judge/

Trên VPS nếu lỡ có node_modules từ Windows:

rm -rf /root/code-judge/node_modules /root/code-judge/apps/*/node_modules

6.4. Cập nhật code sau này

Chạy lại sync từ Windows, rồi trên VPS:

cd /root/code-judge
sed -i 's/\r$//' scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x scripts/isolate_stub.sh scripts/sudo_stub.sh
chmod +x deploy/production-up.sh
./deploy/production-up.sh

Chi tiết từng lệnh: UPDATE-DEPLOY.html.

7. File .env.production trên VPS

6.1. Tạo và chỉnh file trên VPS

Trên VPS, trong thư mục repo:

cd /root/code-judge
cp .env.production.example .env.production
nano .env.production

Hoặc chỉnh trên Windows rồi upload:

cd C:\Users\ADMIN\Documents\GitHub\code-judge
scp .env.production root@VPS_IP:/root/code-judge/.env.production

Bảo vệ quyền file:

chmod 600 /root/code-judge/.env.production
Git không commit .env.production (đã có trong .gitignore). Mỗi lần đổi secret hoặc URL public, sửa file trên VPS rồi chạy lại ./deploy/production-up.sh. Đổi NEXT_PUBLIC_CORE_URL bắt buộc build lại image web (script up đã có --build).

6.2. Sinh secret ngẫu nhiên trên VPS

# JWT (32+ ký tự hex)
openssl rand -hex 32

# Judge0 Rails secret (64+ ký tự hex)
openssl rand -hex 64

# Mật khẩu Postgres / MinIO (ví dụ 24 ký tự base64)
openssl rand -base64 24

Dán giá trị vào đúng dòng trong .env.production.

6.3. Hai kịch bản URL

A — Có domain + DNS

Trỏ bản ghi A:

  • your-domain.com → VPS_IP
  • api.your-domain.com → VPS_IP
FRONTEND_URL=https://your-domain.com,https://www.your-domain.com
NEXT_PUBLIC_CORE_URL=https://api.your-domain.com
COOKIE_DOMAIN=.your-domain.com
MINIO_PUBLIC_BASE_URL=https://your-domain.com
GOOGLE_CALLBACK_URL=https://api.your-domain.com/auth/google/callback

COOKIE_DOMAIN có dấu chấm đầu (VD .code-judge.io.vn) để cookie JWT dùng chung giữa your-domain.com và api.your-domain.com.

MINIO_PUBLIC_BASE_URL = domain web (HTTPS), không :9000 — nginx proxy path /codejudge/ tới MinIO.

User mở web: https://your-domain.com (sau khi gắn HTTPS — mục 11).

B — Chỉ có IP (test nhanh)

FRONTEND_URL=http://203.0.113.10
NEXT_PUBLIC_CORE_URL=http://203.0.113.10:8080
Dịch vụURL trình duyệt / client
Webhttp://VPS_IP/ (nginx port 80)
APIhttp://VPS_IP:8080/ (nginx port 8080)
Đừng dùng localhost trong FRONTEND_URL hay NEXT_PUBLIC_CORE_URL — user truy cập từ máy khác sẽ lỗi CORS / API.

11. HTTPS (Let's Encrypt)

Nginx trong container map 80/443 (web) và 8080/443 (api). Cert thường mount vào container (xem deploy/nginx/ và deploy/README.md).

Sau khi có HTTPS, cập nhật .env.production:

FRONTEND_URL=https://code-judge.io.vn,https://www.code-judge.io.vn
NEXT_PUBLIC_CORE_URL=https://api.code-judge.io.vn
COOKIE_DOMAIN=.code-judge.io.vn
MINIO_PUBLIC_BASE_URL=https://code-judge.io.vn

Trên VPS — áp dụng env và rebuild web (URL API nhúng lúc build):

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate core-api
docker compose -f docker-compose.production.yml --env-file .env.production build web --no-cache
docker compose -f docker-compose.production.yml --env-file .env.production up -d web nginx
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -t
docker compose -f docker-compose.production.yml --env-file .env.production exec nginx nginx -s reload

Kiểm tra:

curl -sI https://code-judge.io.vn | head -5
curl -sI https://api.code-judge.io.vn/tags | head -5
Chỉ HTTP + IP (lab / nội bộ)

Dùng kịch bản mục env-scenarios B — không cần COOKIE_DOMAIN subdomain; không dùng MINIO_PUBLIC_BASE_URL HTTPS.

13. Xử lý lỗi thường gặp

Không SSH được

  • Kiểm tra security group / firewall nhà cung cấp (port 22).
  • Đúng user ubuntu và IP public.

permission denied khi chạy docker

sudo usermod -aG docker $USER
# logout SSH, login lại

Set POSTGRES_PASSWORD in .env.production

Thiếu biến bắt buộc — mở .env.production và set đủ các dòng có ?Set trong compose.

Web lỗi API / CORS / Mixed Content

  • NEXT_PUBLIC_CORE_URL = URL API từ trình duyệt (VD https://api.domain.com), không http://core-api:3000.
  • FRONTEND_URL khớp origin web (kể cả www nếu dùng).
  • Rebuild web sau khi sửa NEXT_PUBLIC_*.
  • MINIO_PUBLIC_BASE_URL phải HTTPS cùng domain web khi trang HTTPS — không :9000.

Login OK nhưng /dashboard hoặc /auth/me 401

cd /root/code-judge
docker compose -f docker-compose.production.yml --env-file .env.production exec core-api printenv COOKIE_DOMAIN FRONTEND_URL
docker compose -f docker-compose.production.yml --env-file .env.production up -d --force-recreate core-api
