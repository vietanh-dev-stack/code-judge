Tổng quan triển khai: Sinh testcase PROJECT

Dự án: code-judge Phiên bản: 1.0 Cập nhật: 2026-05-18 As-built overview

Tài liệu này mô tả những gì đã được triển khai trong repo để hỗ trợ sinh hidden tests cho bài ProblemMode.PROJECT (backend / frontend / fullstack), từ kế hoạch HTML ban đầu đến prompt AI, validator, API test và tab UI trên /test.

Bối cảnh và mục tiêu

Hệ auto-grader PROJECT dự kiến: học viên nộp zip → MinIO → BullMQ → Worker mount student_code + hidden_tests (read-only) → chạy npm test / e2e → trả JSON điểm (score, stats, details[]).

Phần đã làm tập trung vào chuẩn bị hidden tests cho giảng viên:

  • LLM phải hiểu đề bài trước (output problemBrief với FR-1, FR-2, …).
  • Sinh bundle file có thể npm ci và chạy test (Jest/supertest hoặc Playwright).
  • Mỗi test trong testManifest gắn requirement, file path, tên test khớp source.
  • Validator từ chối output thiếu coverage hoặc không nhất quán trước khi coi là hợp lệ.
Khác với bài ALGO (đã có sẵn): ALGO dùng TestCase.input / expectedOutput text + Judge0. PROJECT dùng cây file tests/**/*.spec.ts lưu trong job / DB meta.

Prompt AI và validator

Hai phase trong một JSON response

PhaseOutputMục đích
1 — Phân tích đề problemBrief summary, stack, functionalRequirements[] (FR-*), assumptions, outOfScope
2 — Sinh test files[], testManifest[], runConfig Bundle chạy được; mỗi test map tới FR; tên test có trong file

Schema output (rút gọn)

{
  "problemBrief": {
    "title": "...",
    "summary": "...",
    "stack": "backend" | "frontend" | "fullstack",
    "functionalRequirements": [
      { "id": "FR-1", "description": "...", "priority": "must" }
    ],
    "assumptions": [], "outOfScope": []
  },
  "testManifest": [
    {
      "testName": "Login API",
      "requirementIds": ["FR-1"],
      "suite": "api",
      "filePath": "tests/api/auth.spec.ts",
      "weight": 5,
      "isHidden": true,
      "rationale": "..."
    }
  ],
  "files": [{ "path": "package.json", "content": "..." }],
  "runConfig": {
    "installCommand": "npm ci",
    "testCommand": "npm test -- --json --outputFile=result.json",
    "resultParser": "jest-json",
    "dockerImage": "node:18-alpine"
  }
}

Validator kiểm tra gì?

  • Mọi FR priority must có ít nhất 1 test trong manifest.
  • testManifest.filePath tồn tại trong files[].
  • testName xuất hiện trong nội dung file (describe/it/test).
  • Path an toàn (không .., không absolute).
  • Có package.json và ít nhất một file *.spec.* / *.test.* / e2e.
  • Gợi ý parser: backend → jest-json; frontend → Playwright config.

Nếu validator fail → parseError gồm danh sách mã lỗi; có thể gửi lại với revision trong DTO.

Lưu TestCase khi generate-and-save (PROJECT)

  • input = testName
  • expectedOutput = JSON meta (filePath, requirementIds, suite, …)
  • isHidden = true (mặc định từ manifest)
  • Full file tree nằm trong AiGenerationJob.structuredOutput.files
