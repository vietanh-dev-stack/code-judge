Lab /test — Kiểm thử sinh testcase PROJECT

Dự án: code-judge Route: /test Cập nhật: 2026-05-18 Test lab doc

Tài liệu mô tả phòng lab tích hợp trên web app để thử nghiệm khả năng AI sinh hidden tests cho bài ProblemMode.PROJECT (backend / frontend / fullstack): form đề bài, 3 sample có sẵn, generate draft, xem bundle file, giải thích theo dòng. Không thay thế wizard admin production — chỉ dùng dev/QA.

1. Giới thiệu

Trang Lab tích hợp nằm tại route Next.js apps/web/app/(main)/test/page.tsx, URL khi chạy local thường là http://localhost:3000/test (port tuỳ cấu hình).

Khác với bài ALGO (stdin/stdout, Judge0), bài PROJECT cần AI trả về:

  • problemBrief — phân tích đề, functional requirements (FR-*)
  • testManifest — danh sách test ẩn (tên, file, FR, weight)
  • files[] — mã nguồn test runnable (Jest, Playwright, package.json, …)
  • runConfig — lệnh install/test, parser JSON, docker image
Tab PROJECT testcase là nơi duy nhất trên UI hiện tại để thử end-to-end luồng sinh bundle này mà không cần tạo problem trên admin wizard.

3. Điều kiện chạy lab

  1. core-api đang chạy, CORS/cookie cho phép web gọi API.
  2. web (apps/web) với NEXT_PUBLIC_CORE_URL trỏ đúng core-api.
  3. Một trong hai API key AI (tuỳ provider chọn trên UI):
    • GOOGLE_GENERATIVE_AI_API_KEY — mặc định UI thường chọn google
    • OPENAI_API_KEY — khi chọn openai
  4. Tài khoản ADMIN đã đăng nhập trên web.

5.1. Generate project draft (luồng chính)

  1. Đăng nhập ADMIN, mở /test → tab PROJECT testcase.
  2. Chọn sample (Backend / Frontend / Fullstack) hoặc chỉnh form Tuỳ chỉnh.
  3. Bấm Generate project draft (hoặc Generate (form hiện tại) ở card sample).
  4. UI gọi aiTestcaseApi.generateProjectDraft(form) → POST /ai-testcase/generate-project-draft.
  5. Core parse JSON LLM, chạy validateProjectTestcaseOutput; trả parsed, validation, parseError nếu có.
  6. Card kết quả hiển thị FR, bảng manifest, nút Preview bundle.

5.2. Test sample (không cần sửa form)

  • Test sample đang chọn — một lần LLM với payload sample tích hợp (backend | frontend | fullstack).
  • Test cả 3 sample — gọi API không truyền sample → backend chạy lần lượt 3 sample (3 lần LLM, có thể vài phút).

Sample được định nghĩa tĩnh trong project-testcase-samples.ts (NestJS library, React Todo, Express+React shop).

5.3. Preview bundle & giải thích theo dòng

Component: ProjectTestcaseFilesPreview.tsx + ProjectTestFileExplanationView.tsx

  1. Sau generate thành công, bấm Preview bundle (N file) — Dialog ~96vw × 92vh.
  2. Sidebar trái: danh sách file (tests/, e2e/, package.json…); chọn file xem <pre> cuộn dọc/ngang.
  3. Khung trên file: manifest test thuộc file đó (nếu có).
  4. Bấm Giải thích file này → POST explain-project-test-file với fileContent, problemSummary, relatedTestsJson.
  5. UI hiển thị giải thích có cấu trúc (không phải đoạn văn thuần):
    • Mã + chú thích — bảng số dòng + mã + cột vai trò từng khối
    • Khối mã — từng dòng X–Y, nhãn, trách nhiệm, snippet
    • Test case — bảng test → dòng → kiểm tra gì
Prompt explain: project-testfile-explain.ts — bắt buộc JSON lineBlocks + testBreakdown; validator kiểm tra số dòng không vượt quá file.

6. API Core gọi từ tab PROJECT

Controller: apps/core-api/src/ai-testcase/ai-testcase.controller.ts

Client web: apps/web/services/ai-testcase.apis.ts — object aiTestcaseApi

MethodPathDùng khiRole
GET /ai-testcase/project-testcase-samples Load tab: meta + DTO mẫu cho 3 nút sample ADMIN
POST /ai-testcase/generate-project-draft Generate project draft (form hiện tại) ADMIN
POST /ai-testcase/test-generate-project-sample Test một sample hoặc cả 3 (body sample optional) ADMIN
POST /ai-testcase/test-generate-project-sample/{backend|frontend|fullstack} Shortcut (không dùng trực tiếp từ UI tab; có thể gọi curl) ADMIN
POST /ai-testcase/explain-project-test-file Nút Giải thích file trong preview ADMIN

Endpoint POST /ai-testcase/generate-and-save ghi DB + job — không được gọi từ tab /test hiện tại; dùng khi tích hợp wizard tạo đề production.

9. Gọi API ngoài UI (curl / Swagger)

Thay $CORE và cookie/header JWT ADMIN. Ví dụ generate draft:

curl -X POST "$CORE/ai-testcase/generate-project-draft" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $JWT" \
  -d '{
    "title": "API Demo",
    "statement": "Mô tả đề...",
    "stack": "backend",
    "framework": "nest",
    "maxTestCases": 6,
    "provider": "google"
  }'

Test sample backend:

curl -X POST "$CORE/ai-testcase/test-generate-project-sample" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $JWT" \
  -d '{ "sample": "backend", "provider": "google" }'

Swagger: tag ai-testcase trên core-api khi bật OpenAPI.
