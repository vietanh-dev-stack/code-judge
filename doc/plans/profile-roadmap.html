Kế hoạch triển khai Profile — Code Judge

Tài liệu mô tả trạng thái hiện tại và lộ trình mở rộng trang profile. Không thay đổi schema.prisma cho đến khi team chốt migration.

Cập nhật: Tháng 5/2026. Phase 1–2 đã triển khai (bảo mật, stats từ Submission). Các tính năng cần cột DB mới được hoãn sang Phase 3+ và mô tả dưới đây.

1. Đã triển khai Done

Backend (apps/core-api)

  • GET /auth/me, GET/PATCH /users/me — response an toàn (không passwordHash)
  • UpdateMeDto — chỉ name, email (chặn tự nâng role / isActive)
  • Avatar: presign → MinIO → confirm (ghi image + imageObjectKey)
  • GET /users/me/stats — bài đã giải, theo độ khó, ngôn ngữ, tỷ lệ đúng, hoạt động gần đây
  • POST /auth/change-password
  • DELETE /users/me — vô hiệu hóa tài khoản (soft: isActive = false)

Frontend (apps/web)

  • Route /dashboard/profile — tab Tổng quan, Thống kê, Cài đặt
  • Bỏ mock data; loading / empty state
  • Settings → ?tab=settings; redirect /settings, /dashboard/settings
  • Sau lưu profile / avatar: refreshUser() từ /auth/me

2. Cần migration Prisma Phase 3

Các tính năng sau yêu cầu thêm cột hoặc bảng. Khi triển khai, tạo migration riêng (ví dụ YYYYMMDD_profile_extended) và cập nhật seed nếu cần.

2.1 Mở rộng model User

model User { // ... các field hiện có ... bio String? location String? isProfilePublic Boolean @default(false) }
FieldMục đíchAPI / UI
bioGiới thiệu ngắn (max ~500 ký tự)PATCH /users/me, hiển thị header profile
locationĐịa điểmPATCH /users/me, header
isProfilePublicCho phép xem profile công khaiCheckbox Cài đặt; GET /users/:id/public

2.2 Profile công khai

  • GET /users/:id/public — @Public(), chỉ trả field công khai (không email)
  • Kèm stats rút gọn (optional): problemsSolved, ngôn ngữ top 3
  • Frontend: /users/[id] hoặc /profile/[id]

2.3 Social links

Hai hướng:

  • JSON trên User: socialLinks Json? — nhanh, ít join
  • Bảng riêng UserSocialLink — chuẩn hóa, dễ validate URL

2.4 Achievements (gamification)

  • Model Achievement + UserAchievement
  • Job/cron hoặc hook sau submission Accepted để unlock
  • Tab Achievements trên profile (đã có UI prototype cũ, có thể tái sử dụng layout)

2.5 Preferences & notifications

  • UserPreferences (JSON hoặc bảng): email contest, class announcements, v.v.
  • Tích hợp sau khi có hệ thống gửi email/notification

2.6 Metrics nâng cao (không cần migration User)

  • Streak, level, best rank toàn hệ thống — cần định nghĩa product + có thể bảng UserStats cache
  • Hoạt động contest/class trong recentActivity — mở rộng UserStatsService

3. Thứ tự triển khai đề xuất

  1. Migration User — bio, location, isProfilePublic + API + UI settings
  2. Public profile page — endpoint + route frontend
  3. Social links — theo quyết định JSON vs bảng
  4. Achievements — schema + rules + tab UI
  5. Notifications preferences — khi có email service
  6. Streak / level — product spec + cache layer

4. Checklist trước khi merge migration

  • Review với team: field nullable, default isProfilePublic = false
  • Chạy npx prisma migrate dev trên staging
  • Cập nhật Postman / OpenAPI
  • Kiểm tra GDPR: export data (Download My Data) nếu bật
  • Regression: admin PATCH /users/:id vẫn dùng UpdateUserDto đầy đủ
